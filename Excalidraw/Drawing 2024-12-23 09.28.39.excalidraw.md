-> event consumed by payment service -> payment service publishes a payment successful event

-> event consumed by kitchen service -> kitchen service periodically publishes events about its progress
-> kitchen service finally generates a quality checked event -> event consumed by order service
-> if order is a delivery, will generate an event for the delivery API to consume for delivery, otherwise mark the order as ready for collection
